    Agile. Гибкая методология. Scrum.
    Суть: заказчик вовлечен в процесс разработка, разбит на итерации, в рамках набор функционала,
    в конце итерации - приемка, отзыв заказчика, возможные изменения. 2 - 4 недели - итерация.
    Итерация меньше - мало времени, мало функционала, много совещаний. В конце итерации новый функционал. Больше - приближается в водопадной.
    Определяется опытным путем. Спринт=итерация.
    Функционал описывается в юзер-сторе - похож на
    С началом спринта собираются юзер сторе , продуктоунер выбирает их для спринта. Потом помещаются в бэк логи, в процессе не может
    быть изменен. В первой части митинга фиксируются задачи.
    Обзязательная часть разработки ежедневные митинги. Ежедневно - по 15 мин. Можно еженедельное собрание.
    Кажный должен ответить на 3 вопроса: делал, буду сегодня, какие виже проблемы - они не решаются, для этого есть скрам мастер.

     Основные группы ролей в скрам.

     свиньи - полностью вовлечен, отвечает за результат, скрам мастер - учитывают мнение
    куры - ост - мнения выслушивают
